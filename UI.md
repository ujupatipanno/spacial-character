# 기호 입력 플러그인 UI 재설계

## 설계 원칙
- **단순성**: 인라인 스타일 없이 Obsidian의 기본 Setting 컴포넌트만 사용
- **직관성**: 사용자가 쉽게 이해하고 사용할 수 있는 구조
- **최소주의**: 기본 기호 세트와 프리셋 기능 제거, 사용자가 직접 관리

## 설정 탭 구조

## 구현 방식

### Setting 컴포넌트 활용
Obsidian의 `Setting` 클래스를 활용하여 각 필드를 구성:

```typescript
// 각 기호마다 3개의 Setting 생성
new Setting(containerEl)
    .setName("ID")
    .setDesc(`기호 #${index + 1} - 검색용 식별자`)
    .addText(text => { /* ID 입력 */ })
    .addExtraButton(button => { /* 위로 */ })
    .addExtraButton(button => { /* 아래로 */ })
    .addExtraButton(button => { /* 삭제 */ });

new Setting(containerEl)
    .setName("기호")
    .addText(text => { /* 기호 입력 */ });

new Setting(containerEl)
    .setName("설명")
    .addText(text => { /* 설명 입력 */ });
```

### 구분선 추가
각 기호 항목 사이에 구분선을 추가하여 시각적으로 구분:
```typescript
containerEl.createEl("hr", { cls: "symbol-divider" });
```

### CSS 최소화
- 기본 Obsidian 스타일 최대한 활용
- 필요한 경우에만 추가 클래스 사용 (구분선, 빈 상태 등)

## 삭제할 기능
1. ✅ 기본 기호 세트 (DEFAULT_SETTINGS.symbols를 빈 배열로)
2. ✅ 프리셋 정의 (PRESETS 객체 삭제)
3. ✅ 프리셋 가져오기 UI (설정 탭에서 제거)
4. ✅ 카드 스타일 (symbol-item-card, symbol-card-* 클래스)
5. ✅ 커스텀 버튼 스타일 (Obsidian 기본 버튼 사용)

## 유지할 기능
1. ✅ 트리거 문자 설정
2. ✅ 퍼지 검색 토글
3. ✅ 하이라이트 토글
4. ✅ 설명 표시 토글
5. ✅ 기호 추가/수정/삭제/이동
6. ✅ 자동완성 제안 목록 스타일 (실제 제안 UI에 필요)

## 장점
- **단순함**: 복잡한 카드 레이아웃 제거
- **일관성**: Obsidian 네이티브 UI와 일치
- **유지보수**: 인라인 스타일 없어 코드 간결
- **접근성**: 표준 Setting 컴포넌트 사용으로 키보드 네비게이션 자동 지원

## 구현 순서
1. ✅ UI.md 작성 (완료)
